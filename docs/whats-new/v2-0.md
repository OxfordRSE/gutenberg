Gutenberg v2.0 is the point where the application becomes a broader teaching platform rather than just an event wrapper around markdown material.

The release introduces first-class self-paced courses, richer teaching context, a real default course catalog, and much smoother authoring workflows for both courses and events.

## Self-paced courses are now a first-class workflow

Courses are no longer placeholders. Gutenberg now supports a real curated course catalog built from grouped material sections, with filtering, enrolment-aware views, progress, and structured course editing.

This makes it practical to use Gutenberg for reusable self-paced learning as well as live teaching.

## Active learning context across the app

Users can now work with an active event or active course. That context is used across:

- course cards and course pages
- event flows
- the sidebar
- material page hints and navigation

This gives the app a much stronger sense of "what the learner is currently doing".

## Real default courses and sync tooling

The default course catalog has been replaced with real proof-of-concept courses based on actual HPCu material. Admins can review and sync those defaults into the database directly from the courses page.

The sync review now shows clearer diffs, including grouped material changes broken out per material group.

## Streamlined editing for courses and events

Course editing now happens around grouped material on a single page. Event editing follows the same direction:

- event groups are edited inline on the main event page
- material can be added, removed, and reordered without separate group-edit navigation
- new events can be created either blank or from a course blueprint

This makes the authoring model far more coherent than the earlier event-only flow.

## Better material navigation and progress surfaces

Material pages can now reflect course and event structure more directly:

- course-aware material hints
- event-aware and course-aware previous/next links
- return-to-course behavior at the end of a course sequence
- improved home-page course surfaces and progress fetching

Together, these changes make the learning flow feel much more connected.

## What this release means

Gutenberg v2.0 does not replace the event model. Instead, it adds a stronger reusable curriculum layer on top of the material and lets events build on that layer when appropriate.
