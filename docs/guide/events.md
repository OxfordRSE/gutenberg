Events are Gutenberg's scheduled teaching deliveries: workshops, short courses, lecture series, and other cohort-based teaching.

## Creating events

Admins can create an event from the event list or home page. There are two starting modes:

- **Blank event** for a fresh event with no copied teaching structure
- **From course** to copy the groups and material from an existing course as a starting blueprint

Both creation modes capture an event start time up front.

## Event editing

The main event page now owns the edit workflow. You no longer need to navigate to a separate event-group edit page just to add material.

The current event editor supports:

- editing event metadata on the main event page
- adding and removing event groups inline
- editing each group's name, summary, location, and times inline
- adding, removing, and reordering material sections inside each group on the same page

This makes event editing consistent with the newer grouped course editor.

## Enrolment and instructor workflows

Events can use:

- an **enrolment key** for student enrolment
- an **instructor key** for enrolling instructors directly
- manual enrolment requests when keys are not used or a student needs approval

Once enrolled, students work through the event material while instructors and admins can inspect cohort progress on the event page.

## Problems and progress

When event groups include material with embedded problems, Gutenberg tracks problem completion against the event. Students see their own progress while instructors can inspect group progress across the cohort.
